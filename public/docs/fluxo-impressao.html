🖨️ WingX — Fluxo de Impressão de Etiquetas

Do momento da venda até a etiqueta impressa na sua mesa

Mercado Livre
⏳
Passo 1a — Pagamento Pendente
Webhook captura payment_required
Antes do pagamento ser confirmado, o ML pode gerar um evento payment_required. O MC salva o pedido com status pendente e envia notificação WhatsApp com badge ⏳.
payment_required WhatsApp ⏳
🛍️
Passo 1b — Pagamento Confirmado
Venda paga no Mercado Livre
Comprador finaliza o pagamento. O ML gera automaticamente uma notificação de pedido pago (orders_v2 / paid). Status atualizado de pendente para pago.
ML API status: paid
Mission Control (VPS)
⚡
Passo 2
Webhook recebe a notificação
O MC recebe o aviso do ML em https://mc.wingx.app.br/api/mercado-livre/webhook, busca os detalhes do pedido + envio e salva no banco de dados.
PostgreSQL ml_pedidos ml_clientes
🖨️
Passo 3
Job de impressão criado na fila
Um registro é criado na tabela print_queue com status queued e um token único de segurança. A etiqueta ainda não foi impressa — aguardando sua aprovação.
print_queue status: queued
WhatsApp — Grupo "Venda ML"
📲
Passo 4
Notificação de venda chega no WhatsApp
O grupo recebe a mensagem com foto do produto, nome do comprador, CPF, telefone, endereço de entrega, endereço da nota fiscal e total da venda.
Evolution API Foto + Dados
🔗
Passo 5
Link de impressão chega logo depois
Uma segunda mensagem chega com o link de aprovação: 🖨️ Imprimir etiqueta → mc.wingx.app.br/api/print-queue/trigger?token=xxxx
Token único HTTPS seguro
👆
Passo 6 — Você age
Você toca no link pelo celular
Ao tocar, o browser abre e confirma: "✅ Impressão solicitada! O agente no notebook vai imprimir em instantes." O job muda de queued → pending.
status: pending
WingX Agent (seu notebook)
🔄
Passo 7
Agente detecta o job pendente
O WingX Agent roda em segundo plano no notebook e faz polling a cada 10 segundos no MC. Ao encontrar um job pending, começa o processo de impressão.
Polling 10s x-agent-key
📄
Passo 8
MC baixa a etiqueta PDF do Mercado Livre
O agente pede o PDF ao MC. O MC usa o token OAuth armazenado (renovado automaticamente) para buscar a etiqueta no ML API /shipment_labels?shipment_ids=...&response_type=pdf e devolve o PDF ao agente.
ML OAuth token Proxy seguro
Impressora USB
🖨️
Passo 9
Etiqueta enviada para a impressora
O agente usa o SumatraPDF (baixado automaticamente) para imprimir o PDF em silêncio — sem janelas, sem popups. Funciona com qualquer impressora USB conectada ao Windows.
SumatraPDF USB / Rede
✅
Passo 10
Job marcado como concluído
O agente confirma ao MC que a impressão foi feita. O status muda para done. Em caso de erro (sem shipment, token expirado), o status muda para error com a mensagem de erro para diagnóstico.
status: done
QR Code — Confirmação de Embalagem
📦
Passo 11
QR Code gerado após impressão
Ao imprimir a etiqueta, o MC gera um QR Code único apontando para /api/print-queue/confirm?token=.... O QR Code fica disponível no dashboard junto ao job.
qr_code_url print_queue
📷
Passo 12 — Você age
Escanear QR Code confirma embalagem
Após colocar a etiqueta na caixa, escaneie o QR Code com o celular. O status muda de done → confirmed, registrando confirmed_at e confirmed_by no banco.
status: confirmed confirmed_at / confirmed_by
📦
Pedido embalado e confirmado!
Fluxo completo: payment_required → paid → print_queue → done → QR scan → confirmed

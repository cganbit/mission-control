Mecanismos de autenticação
JWT

Cookie httpOnly mc_token — login via /login. Dashboard e todas as rotas autenticadas.

JWT admin

JWT com role admin. Gestão de usuários, conectores, remoção de squads.

x-agent-key

Header x-agent-key: PRINT_AGENT_KEY — agente impressor WingX local.

x-worker-key

Header x-worker-key: WORKER_KEY — cron de refresh de tokens e catálogo ML.

token na URL

Query ?token= único por job — QR confirm e trigger do WhatsApp. Sem login.

público

Sem autenticação. Webhook ML, health check, OAuth callback, confirm/trigger por token.

⚠️ Notas e ambiguidades
Webhook ML sem HMAC

POST /api/mercado-livre/webhook não valida assinatura HMAC. Aceita qualquer POST com body válido. Risco de spoofing se a URL vazar.

OAuth callback público

GET /api/mercado-livre/oauth/callback sem verificação de sessão. Mitigação: TTL curto do code do ML.

Label — 3 mecanismos de auth

/api/print-queue/[id]/label aceita agent-key, queue-key ou JWT. Via queue-key (celular) só serve PDF já armazenado — não busca do ML.

✅ WORKER_KEY — fallback removido (27/03/2026)

Hardcoded 'catalogo-worker-2026' removido dos 3 endpoints. Agora exige a env var explicitamente. Verificar que WORKER_KEY está setada no VPS.

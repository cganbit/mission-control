⚠️ Precisa Ajustar
Admin não pode assignar donoUI gap
No modal "Adicionar Conta", admin não vê campo para escolher owner_user_id. A conta é sempre criada como pertencente ao admin — impede delegar contas para members.
falta campo selectAPI suporta
Contas sem dono são invisíveis para memberslógica
GET /accounts filtra por owner_user_id = session.sub para members. Se owner_user_id for NULL (conta importada), nenhum member vê. Admin vê, mas member não consegue reclamar a conta.
falta "claim" endpoint
Sender único compartilhadoWhatsApp
Todos os grupos de notificação recebem mensagem do MESMO número WhatsApp (Evolution API do sistema). Num cenário multi-cliente, o cliente B vê mensagens vindas do número do cliente A.
Evolution instance única
Validação de notification_groupUX
Campo aceita qualquer string. Não valida formato remoteJid (ex: 5511@s.whatsapp.net vs grupo@g.us). Erros silenciosos se o formato estiver errado.
sem feedback visual
PATCH não atualiza nicknameAPI gap
Se o nickname do vendedor mudar no ML, a tabela ml_account_configs não é atualizada automaticamente. Seria útil sincronizar na próxima chamada ao webhook.
🔮 Roadmap SaaS
Evolution API por clienteSaaS
Cada cliente conecta seu próprio WhatsApp via QR Code. Sistema cria nova instância Evolution API por tenant. Mensagens saem do número do próprio cliente.
instância por tenant
Onboarding com QR CodeUX
Flow de setup: cliente cria conta → conecta WhatsApp via QR → adiciona contas ML → sistema começa a notificar pelo número dele. Sem intervenção manual do admin.
self-serve
Billing por conta MLmonetização
Cobrar por número de contas ML ativas, volume de notificações enviadas ou jobs de impressão disparados. Integrar com Stripe ou Pagar.me.
ML OAuth por clienteintegração
Cada cliente autentica suas próprias contas ML via OAuth2 (App ML). Tokens armazenados por tenant. Hoje tokens são do admin (ml_tokens_json global).
OAuth2 ML
Dashboard por tenantisolamento
Cada cliente vê apenas seus dados: vendas, fila de impressão, clientes, pedidos. Row-level security no PostgreSQL por tenant_id.
RLS PostgreSQL
